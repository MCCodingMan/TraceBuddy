# TraceBuddy UI 设计规范

## 品牌与主题
- 调色方案：主色 `#00D4FF`（Cyan），次色 `#8B5CF6`（Purple），强调色 `#FF00FF`（Magenta）
- 背景：深色渐变，层级背景使用半透明玻璃质感卡片
- 语气：科技、专业、现代

## 色彩系统
- 文字：主文本 `#FFFFFF`，次文本 `rgba(255,255,255,0.8)`，弱化文本 `rgba(255,255,255,0.6)`
- 边框：默认 `rgba(0,212,255,0.3)`，悬停 `rgba(0,212,255,0.6)`，焦点 `#00D4FF`
- 状态：
  - 成功 `#10B981`
  - 警告 `#F59E0B`
  - 错误 `#EF4444`
  - 信息 `#3B82F6`

## 排版
- 字体族：`Rajdhani`, `Inter`, `system-ui`, `sans-serif`
- 标题层级：
  - H1 28–32px，粗体
  - H2 22–24px，半粗体
  - H3 18–20px，半粗体
- 正文：14–16px，行高 1.5–1.7
- 单行代码与数值：使用等宽体（font-mono）

## 间距与网格
- 间距刻度：4 / 8 / 12 / 16 / 24 / 32 px
- 卡片内边距：`16–24px`
- 栅格：移动 1 列，平板 2 列，桌面 3–4 列（按内容密度调节）

## 组件样式
- 卡片（`.tech-card`）：半透明背景、圆角 12–16px、轻微发光、顶部渐变分隔线、悬停上浮与边框增强
- 按钮（`.tech-button`）：
  - Primary：青紫渐变、白字、悬停强化阴影与轻微上浮
  - Secondary：半透明深色背景、浅色文字与边框
  - Ghost：透明背景、浅色边框，用于次要操作
  - 尺寸：高度 ≥ 48px，圆角 8–12px，内边距 12×24px
- 输入框（`.tech-input`）：深色半透明背景、统一圆角与边框、焦点蓝色发光环、占位符弱化
- 标签与徽章：根据状态使用对应色彩，尽量配合等宽字体呈现数值

## 交互与动效
- 悬停：轻微上浮（1–2px）、阴影增强、颜色亮度提升
- 焦点：以蓝色边框与浅发光环提示
- 过渡：200–300ms，缓动 `ease` / `ease-in-out`
- 进场：`fade-in`（位移 4px，时长 300ms）

## 响应式规则
- 断点策略：
  - ≤640px：单列布局、扩大触控目标、隐藏不关键的装饰
  - 641–1024px：双列布局、收敛内边距与边框
  - ≥1025px：多列布局、完整视觉元素
- 表格：在小屏下保留关键列（时间、API、状态），细节通过展开查看

## 视觉层次
- 层级由大小、颜色、间距共同构建：
  - 重要信息（标题、关键指标）：更大字号与更高对比度
  - 操作控件：统一按钮风格，主次分明
  - 卡片：分组信息、留白充足，避免拥挤

## 组件用法约定
- 登录页：输入框使用 `.tech-input`，提交按钮使用 `.tech-button` Primary
- 过滤器、统计卡片、表格容器：包裹 `.tech-card`，必要处添加 `hover-lift` 与 `fade-in`
- 表格行：可点击区域展开详情，局部操作阻止事件冒泡

## 无障碍与可访问性
- 对比度：保留高对比度媒体查询调优
- 键盘：表格行支持 Enter/Space 展开；按钮与表单拥有可见焦点样式
- 触控：移动设备控件高度 ≥ 44px

## 浏览器兼容性
- 目标：最新两代 Chrome、Safari、Firefox；Chromium Edge 最新版
- 特性：CSS 变量、`backdrop-filter`、渐变与动画在现代浏览器表现优良；为老版本保持基本布局与交互，无关键功能退化
- 验证方案：
  - 本地预览检查布局与交互（Chrome/Safari/Firefox）
  - 表单焦点、按钮悬停、卡片动效一致性核对
  - 小屏模拟（DevTools）核验响应式断点
